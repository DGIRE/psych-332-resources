Interactive Visualization

UMAP (Uniform Manifold Approximation and Projection)

New to UMAP? Read the UMAP Explained tutorial for an overview of how it works, what the settings do, and how to avoid common misinterpretations.

UMAP Settings

n_neighbors controls the size of the local neighborhood used to learn the manifold structure. Low values (5–10) preserve fine local detail and reveal tight sub-clusters. Higher values (30–50) capture more global structure but may merge nearby groups. This is the most important UMAP parameter — start with 15 and adjust from there.

min_dist controls how tightly UMAP packs points together in the low-dimensional space. Values near 0 produce dense, well-separated clusters. Larger values (0.5–1.0) spread points more evenly, preserving broader topological structure at the expense of cluster compactness.

n_epochs sets how many optimization passes the algorithm runs. More epochs produce a more refined embedding but take longer. 200–500 is typical; increase if the layout still looks unstable or crowded.

Spread works together with min_dist to control the scale of the embedding. Higher spread values push clusters further apart. A spread of 1.0 is the default; increasing to 2–3 can help when clusters are tightly overlapping.

Clustergram

New to hierarchical clustering? Read the Hierarchical Clustering Explained tutorial for an overview of how it works, what linkage methods do, and how to interpret dendrograms and clustergrams.

Clustering Settings

Distance Metric determines how difference between two items is measured. Euclidean is straight-line distance in feature space. Manhattan sums absolute differences along each axis, emphasizing uniform deviations. Correlation measures similarity in expression patterns regardless of magnitude. Cosine compares the angle between feature vectors, ignoring overall scale.

Linkage Method determines how distance between clusters is computed when merging. Average uses the mean distance between all pairs across two clusters, producing balanced trees. Single uses the shortest distance, creating elongated “chaining” clusters. Complete uses the longest distance, producing compact spherical clusters. Ward’s minimizes within-cluster variance at each merge, often producing the most evenly sized clusters.
