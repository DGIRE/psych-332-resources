Interactive Visualization

t-SNE (t-distributed Stochastic Neighbor Embedding)

t-SNE Settings

Perplexity controls the effective number of neighbors each point considers. Low values (5–10) emphasize very local structure, revealing tight sub-clusters. High values (30–50) capture broader patterns but may blur fine detail. A good starting range is 5–50, roughly proportional to dataset size.

Learning Rate controls the step size during gradient descent optimization. Too low and the embedding may get stuck in a poor local minimum; too high and points oscillate without converging. Values between 100–500 work well for most datasets.

Iterations sets how many optimization steps are run. More iterations give the algorithm longer to settle into a stable embedding. 500–1000 is typical; increase if the plot still looks noisy.

Early Exaggeration multiplies all pairwise affinities during the first 100 iterations, forcing clusters apart before fine-tuning. Higher values create wider initial separation between clusters. The default of 4 works well in most cases; increase to 8–12 if clusters appear overlapping.

Interactive Visualization

Clustergram

Clustering Settings

Distance Metric determines how difference between two items is measured. Euclidean is straight-line distance in feature space. Manhattan sums absolute differences along each axis, emphasizing uniform deviations. Correlation measures similarity in expression patterns regardless of magnitude. Cosine compares the angle between feature vectors, ignoring overall scale.

Linkage Method determines how distance between clusters is computed when merging. Average uses the mean distance between all pairs across two clusters, producing balanced trees. Single uses the shortest distance, creating elongated “chaining” clusters. Complete uses the longest distance, producing compact spherical clusters. Ward’s minimizes within-cluster variance at each merge, often producing the most evenly sized clusters.
